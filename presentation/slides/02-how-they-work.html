EVM, yaşam döngüsü, gas ücretleri ve Solidity

Now let's look under the hood. How does a smart contract actually run? What makes it tick?

İki Hesap Türü

Ethereum'da her şey hesaplar aracılığıyla çalışır. İki farklı türü anlamak, akıllı kontratların temelini kavramak demektir.

Dışarıdan Sahipli Hesap (EOA)

Özel anahtar ile kontrol edilir
İşlem başlatabilir
Ücretsiz oluşturulur
İçinde kod yoktur

Kontrat Hesabı

Kod tarafından kontrol edilir
Sadece işlemlere yanıt verir
Oluşturmak gas harcar
Kod + depolama içerir

 Her Hesabın 4 Alanı

Nonce (gönderilen işlem sayısı) • Balance (wei cinsinden bakiye) • CodeHash (EVM kodu hash'i) • StorageRoot (depolama kökü)

There are two types of accounts on Ethereum. EOAs are controlled by private keys and can initiate transactions — these are regular user wallets. Contract accounts are controlled by code and can only respond to transactions, never initiate them. Every account has four fields: nonce, balance, code hash, and storage root.

Ethereum Sanal Makinesi (EVM)

EVM, akıllı kontratların çalıştığı ortamdır — küresel, merkeziyetsiz bir bilgisayar olarak düşünebilirsiniz.

Her Düğümde Çalışır

Her Ethereum düğümü EVM'yi eş zamanlı çalıştırır ve sonuç üzerinde uzlaşı sağlar.

İzole Ortam

Kontratlar izole bir şekilde çalışır. İnternete, dosya sistemine veya diğer süreçlere erişemezler.

Deterministik

Aynı girdi her zaman aynı çıktıyı üretir. Rastgelelik yok, sürpriz yok.

Gas-Sınırlı Turing Tam

Döngüler, koşullar ve veri depolama destekler — ancak her işlem gas ile sınırlıdır, bu sayede sonsuz döngüler imkansızdır.

The EVM is like a global, decentralized computer. Every Ethereum node runs it simultaneously. It's sandboxed — contracts can't access the internet or file system. It's deterministic — same input always gives same output. And it's Turing complete — meaning it can compute anything.

 Önemli

Yayınlandıktan sonra kontrat, blokzincir üzerinde benzersiz bir adres alır. Herkes bu adrese işlem göndererek kontratın açık fonksiyonlarıyla etkileşime geçebilir.

The lifecycle is: Write the contract in Solidity, compile it to bytecode the EVM can understand, deploy it to the blockchain (which costs gas), then anyone can interact with it by calling its functions, and finally you can verify all transactions on a block explorer like Etherscan.

Gas Ücretleri

Gas, Ethereum üzerinde işlemleri yürütmek için gereken hesaplama gücünü ölçen birimdir. Birim: gwei (1 ETH = 10⁹ gwei).

Gerçek Hayat Analojisi

Arabanız gitmek için yakıta ihtiyaç duyar

Daha fazla mesafe = daha fazla yakıt

Yakıt fiyatı piyasaya göre değişir

Yakıt biterse = araba durur, ama mesafeyi korusunuz

Ethereum Gas

Kontratlar çalışmak için gas'a ihtiyaç duyar

Daha fazla hesaplama = daha fazla gas

Gas fiyatı talebe göre değişir

Gas biterse = işlem geri alınır ama gas hâlâ harcanır

EIP-1559 Ücret Modeli

Toplam = Base Fee + Priority Fee
Base fee protokol tarafından belirlenir ve yakılır. Priority fee (bahşiş) doğrulayıcılara gider.

Layer 2 Çözümleri

Arbitrum, Optimism, Base gibi L2'ler işlemleri Ethereum dışında yürütüp ana zincire gönderir — gas maliyetini büyük ölçüde düşürür.

Gas is like fuel for your car. Your car needs fuel to drive — the more distance, the more fuel. Similarly, contracts need gas to execute — the more computation, the more gas. Gas prices vary by network demand, and if you run out of gas, the transaction reverts. Gas exists to prevent spam and infinite loops.

Basit Bir Akıllı Kontrat

SimpleStorage.sol
// ① State
contract SimpleStorage {
    uint256 private storedValue;

    // ② Write
    function set(uint256 _v)
        public
    {
        storedValue = _v;
    }

    // ③ Read
    function get()
        public view
        returns (uint256)
    {
        return storedValue;
    }
}
1

Durum Değişkeni — Blokzincirde kalıcı olarak saklanan veri. private diğer kontratların okumasını engeller, ancak veri zincir üzerinde herkes tarafından görülebilir.

2

Yazma Fonksiyonu — Blokzincir durumunu değiştirir. Bir işlem gerektirir ve gas harcar.

3

Okuma Fonksiyonu — Sadece veri okur. Dışarıdan çağrıldığında (eth_call) ücretsizdir. Zincir üstü bir işlemden çağrılırsa gas harcar.

Here's the simplest possible smart contract — it stores a number. It has a state variable that lives on the blockchain, an event that notifies when the value changes, a set function that writes data (costs gas), and a get function that reads data (free). This is the foundation of all smart contracts.

ABI — Köprü

Solidity kodu derlendiğinde iki çıktı üretilir:

Bytecode

EVM'nin çalıştırdığı düşük seviye makine kodu. Blokzincire yayınlanan budur. İnsanlar tarafından okunamaz.

ABI (Application Binary Interface)

Kontratın fonksiyonlarını, parametrelerini ve dönüş tiplerini tanımlayan JSON şeması. Web2 ile Web3 arasındaki köprüdür.

SimpleStorage ABI (kısaltılmış)
[{
  "name": "set",
  "type": "function",
  "inputs": [{
    "name": "_value",
    "type": "uint256"
  }],
  "stateMutability": "nonpayable"
}]
When you compile Solidity code, you get two outputs. The bytecode is the low-level machine code that the EVM executes — it's what actually gets deployed to the blockchain. The ABI is a JSON specification that describes the contract's interface: what functions are available, what parameters they take, and what they return.

Neden ABI Önemli?

ABI olmadan uygulamalar kontratla nasıl konuşacağını bilemez. Farkı görelim:

ABI ile

// ethers.js — okunabilir ve temiz
const contract = new Contract(
    address, abi, signer
);
await contract.set(42);
const val = await contract.get();

ABI olmadan

// Ham calldata — okunamaz
await signer.sendTransaction({
    to: address,
    data: "0x60fe47b1000000000000"
          "00000000000000000000"
          "0000000000000000002a"
});

Fonksiyon Seçici

ABI, fonksiyon adını 4-byte seçiciye dönüştürür. set(uint256) → 0x60fe47b1

Frontend Entegrasyonu

React, Vue gibi framework'ler ABI sayesinde kontrat fonksiyonlarını doğrudan çağırabilir.

This slide shows the practical difference ABI makes. With ABI, you call contract.set(42) — clean, readable, type-safe. Without ABI, you'd need to manually encode raw hex calldata — the function selector plus ABI-encoded parameters. The function selector is the first 4 bytes of the keccak256 hash of the function signature. Libraries like ethers.js and web3.js handle all of this for you, but they need the ABI to do it.
